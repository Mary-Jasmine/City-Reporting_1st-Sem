User Management

1 How do I create a new user account for barangay officials?

To create a new user account:

1
Navigate to the Users section from the admin dashboard
2
Click the "Add New User" button
3
Fill in the required information:
  • Full Name
  • Email Address
  • Contact Number
  • Barangay Assignment
  • User Role (Barangay Official, Viewer, etc.)
4
Click "Create Account"
The system will automatically generate a temporary password that will be sent to the user's email.
2 How do I reset a user's password?

To reset a user's password:

1
Go to Users section and find the user in the list
2
Click the "Actions" button next to the user
3
Select "Reset Password" from the dropdown menu
4
Confirm the action when prompted
The user will receive an email with a password reset link. This action invalidates their current password immediately.
3 What user roles are available and what permissions do they have?

The system supports four user roles with different permission levels:

1
Super Administrator
  • Full system access
  • Can manage all users
  • System configuration rights
  • Access to all reports and analytics
2
Municipal Administrator
  • Manage barangay users
  • Review and verify incidents
  • Generate municipal reports
  • Create announcements
3
Barangay Official
  • Submit incident reports
  • View barangay-specific data
  • Update incident status
  • Access hotlines and resources
4
Viewer
  • Read-only access
  • View public announcements
  • Access emergency resources
  • No submission or editing rights

Incident Management

4 How do I verify and approve reported incidents?

Incident verification process:

1
Go to Incidents section
2
Click on a pending incident to view details
3
Review the reported information and attached files
4
Use the status dropdown to:
  • Approve - Incident is valid and verified
  • Require More Info - Request additional details
  • Reject - Incident is invalid or duplicate
5
Add verification notes if necessary
6
Click "Update Status"
5 What should I do with duplicate incident reports?

When encountering duplicate reports:

  1. Identify duplicates: Check for similar incidents reported within the same timeframe and location
  2. Merge information: Consolidate all relevant details from duplicate reports into the primary incident
  3. Update status: Mark duplicates as "Merged" with reference to the primary incident ID
  4. Notify reporters: Optionally send a notification to inform about the merge
  5. Document: Add a note explaining the merge for audit purposes
The system automatically suggests potential duplicates based on location proximity and time similarity.
6 How do I escalate an incident to higher authorities?

To escalate an incident:

1
Open the incident details
2
Click the "Escalate" button
3
Select the escalation level:
  • Municipal Level - For barangay-wide issues
  • Provincial Level - For multi-municipality issues
  • Regional Level - For major emergencies
4
Add escalation notes and required actions
5
Click "Confirm Escalation"
Escalated incidents trigger automatic notifications to the relevant authorities and require immediate attention.

Analytics & Reports

7 How do I generate monthly incident reports?

Monthly report generation:

1
Navigate to Analytics dashboard
2
Click on "Generate Report"
3
Select report parameters:
  • Report Type: Monthly Summary
  • Month and Year
  • Barangay(s) to include
  • Incident types to analyze
4
Choose output format:
  • PDF Document
  • Excel Spreadsheet
  • Printable HTML
5
Click "Generate"
Reports are automatically saved to the system archive and can be accessed anytime from the Reports section.
8 How can I identify high-risk areas using the heatmap?

Using the heatmap to identify risk areas:

  1. Access Heatmap: Go to the Heatmap section from admin dashboard
  2. Apply Filters: Filter by date range and incident type
  3. Interpret Colors:
    • Red Areas: High incident density - Immediate attention needed
    • Yellow Areas: Medium incident density - Monitor closely
    • Green Areas: Low incident density - Standard monitoring
  4. Analyze Patterns: Look for clusters and trends over time
  5. Generate Insights: Click on areas to view detailed statistics
The heatmap updates in real-time as new incidents are reported and verified.

System Configuration

9 How do I update emergency hotline numbers?

Updating hotline information:

1
Go to Hotlines section
2
Click on the agency you want to edit
3
Update the contact information:
  • Phone Numbers (Mobile/Landline)
  • Agency Name
  • Emergency Type
  • Operating Hours
4
Click "Save Changes"
Changes to hotline numbers are immediately reflected in the user interface. Consider sending a system announcement about important changes.
10 How do I schedule system maintenance?

To schedule system maintenance:

1

2

3
Click "Schedule Maintenance"
4
Configure maintenance details:
  • Start Date and Time
  • Expected Duration
  • Maintenance Type (Database, Updates, etc.)

5
Create an automatic announcement to notify users
6
Click "Schedule"

Security & Access Control

11 How do I enable two-factor authentication for admin accounts?

Enabling 2FA for administrators:

1
Go to Security Settings
2
Select "Two-Factor Authentication"
3
Choose 2FA method:
  • Email Verification - Code sent to registered email
  • SMS Verification - Code sent to mobile number
  • Authenticator App - Use Google Authenticator or similar
4
Configure settings for admin accounts
5
Click "Enable 2FA"
Two-factor authentication significantly enhances account security. All admin accounts will be required to set up 2FA on next login.
12 How do I review system logs for suspicious activities?

Monitoring system logs:

1
Navigate to Security Settings
2
Select "System Logs"
3
Apply filters to review specific activities:
  • Date Range
  • User ID or Name
  • Action Type (Login, Report, Update, etc.)
  • IP Address
4
Look for suspicious patterns:
  • Multiple failed login attempts
  • Unusual access times
  • Unauthorized access attempts
  • Multiple account lockouts
5
Export logs for further analysis if needed
Regular log review is essential for maintaining system security. Consider scheduling weekly security audits.
